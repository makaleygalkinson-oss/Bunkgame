# Исправление проблемы с Node.js в PATH

## Проблема
Команда `node` не распознается в PowerShell, хотя Node.js установлен.

## Быстрое решение (для текущей сессии)
В текущем окне PowerShell уже добавлено в PATH. Команда `node` должна работать.

## Постоянное решение

Чтобы Node.js работал во всех новых терминалах, нужно добавить его в системную переменную PATH:

### Способ 1: Через графический интерфейс (рекомендуется)

1. Нажмите `Win + R`, введите `sysdm.cpl` и нажмите Enter
2. Перейдите на вкладку **"Дополнительно"**
3. Нажмите кнопку **"Переменные среды"**
4. В разделе **"Системные переменные"** найдите переменную `Path` и нажмите **"Изменить"**
5. Нажмите **"Создать"** и добавьте: `C:\Program Files\nodejs`
6. Нажмите **"ОК"** во всех открытых окнах
7. **Перезапустите VS Code** (закройте и откройте заново)

### Способ 2: Через PowerShell (от имени администратора)

1. Откройте PowerShell **от имени администратора** (правой кнопкой мыши → "Запуск от имени администратора")
2. Выполните команду:
   ```powershell
   [Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\nodejs", [EnvironmentVariableTarget]::Machine)
   ```
3. **Перезапустите VS Code**

### Проверка

После перезапуска VS Code выполните:
```powershell
node --version
npm --version
```

Обе команды должны работать без ошибок.

## Примечание

Если Node.js установлен в другом месте, замените `C:\Program Files\nodejs` на правильный путь к папке, где установлен Node.js.
